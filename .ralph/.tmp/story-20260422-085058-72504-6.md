### LLR-006: Add Language-Aware Local Model Routing
Status: in_progress
Depends on: LLR-003

Description:
Add routing policy that can choose different local models/endpoints for English, Traditional Chinese, and bilingual outputs while preserving existing explicit model behavior.

Acceptance Criteria:
- [ ] Add config shape for local routing defaults such as English model, Traditional Chinese model, bilingual model, and fallback local model.
- [ ] If a user explicitly supplies a model, preserve existing behavior and do not override it.
- [ ] When local routing is enabled and no explicit model is supplied, choose the model based on requested output language.
- [ ] Traditional Chinese output should be able to prefer a Qwen-class local model by config/default profile.
- [ ] English output should be able to prefer a Gemma/Llama-class local model by config/default profile.
- [ ] Add deterministic unit tests for routing choices and explicit-model precedence.
- [ ] Run targeted tests plus pnpm -s typecheck.
